# 批改示範:好 vs 壞會議摘要(同一案例對照)

> 用途:內訓最快「對號入座」的教材。**同一場會議,C1 爛版 vs C5 好版**,逐項用 `03 評估表` 打分,把對角線標出來。
> 註:案例為 AI-Simulated,公司/地名皆虛構(ABC公司 / BBB ERP / 城市X),非真實客戶。

---

## 0. 案例背景(給學員的原始輸入)

> ABC公司(城市X 的傳統機械加工廠,員工約 120 人)。業務經理在會議上抱怨:「我們**報價太慢**,客戶等三天還沒拿到報價,單子被別人搶走。老闆一直唸,但我們也不知道怎麼快。」會中還提到:最近一個大客戶要求「能不能線上看報價進度」;隔壁同業 DEF 去年導入了某系統後接單變快。

---

## 1. ❌ 壞版本(C1 紀錄型 —— 多數讓主管不滿意的長相)

> **ABC公司會議紀錄**
> 1. 客戶說報價太慢,三天還沒給,單子被搶。
> 2. 老闆有抱怨。
> 3. 大客戶想線上看報價進度。
> 4. 同業 DEF 有導入系統。
> 5. 待辦:回去研究報價系統,下週回報。

**評分(03 評估表)**:
| 項目 | 分 | 說明 |
|---|---|---|
| 紀錄完整度 | 1 | 有記但混亂、無結構 |
| 痛點理解 | 0 | 只抄原話「報價太慢」|
| 未來規劃 | 0 | 沒建議 |
| 競爭思考 | 1 | 提到 DEF,但沒分析 |
| 市場/環境 | 0 | 無 |
| 商業判斷 | 0 | 無 |
| **小計** | **2/12** | **對角線:0 條 → 封頂 C1** |

**診斷**:停在「客戶×過去/現在」兩格,把客戶原話搬運一遍。**填了格子、沒有思考。**

---

## 2. ✅ 好版本(C5 顧問型 —— 同一場會議)

> **ABC公司 顧問式會議摘要**
>
> **1. 客戶原話(事實層)**:報價要三天、被搶單、老闆唸;大客戶要線上看進度;同業 DEF 導入系統後接單變快。
>
> **2. 客戶真正痛點(表面→真因)**:表面是「報價慢」,真因是**資料分散(料號/工時/成本在不同人腦袋與 Excel)+ 責任不清 + 沒有標準報價流程**。慢只是症狀。(Ladder:這是會中「不知道怎麼快」透露的訊號,非腦補。)
>
> **3. 現在需求**:他現在要的不只是「快」,是**老闆的掌握感 + 減少對老師傅的人力依賴**(JTBD:他想「完成」的是「不被老闆唸、也不怕老師傅請假就報不出價」)。
>
> **4. 未來顧問建議 + 沒說出口的需求** ⭐:
> - 近期(H2):建標準化報價流程 + 報價資料集中。
> - **沒說出口(H3)**:那位大客戶要「線上看進度」——這其實預告了**未來客戶會要求「報價透明化/自助查詢」**,不只 ABC 內部要快。建議現在就把架構留好「對外可開放查詢」的接口,不要只做內部加速,**否則一年後又要重做**。
>
> **5. 競爭與環境**:
> - 對手 DEF **現在**已用系統接單變快(對手現在的優勢);但多數同業**過去**靠老師傅經驗報價(環境的歷史結構)——這代表**先標準化的人會拉開差距**。
> - 環境:製造業缺工是不可逆趨勢(PESTEL-社會)→「降低對老師傅依賴」會越來越值錢。
>
> **6. 我們(TigerAI)怎麼切入**:
> - 能解:報價流程標準化 + 資料集中 + 報價輔助。不能解(先不碰):ERP 全面替換。
> - 賺法:導入費 + 月費維運;**產品化**成「機械加工報價模板」避免每家從零客製(避免客製地獄)。
> - 壁壘:累積各廠報價 know-how → 模板越用越準。
>
> **7. 下一步**:王經理提供 3 張歷史報價單(本週);我方兩週內出「報價流程現況圖 + 改善路線」。風險:資料散落、老師傅配合度。
>
> **⭐ 對角線(明寫)**:**對手 DEF「現在」的盲點**(只做內部接單快,沒做對外透明)× **客戶「未來」會浮現的需求**(報價對外自助查詢)× **環境「過去」已發生的結構轉變**(缺工、客戶要透明)→ 所以建議:不要只追「報價變快」,要一步到位做「**可對外開放的標準化報價平台**」,直接跳過 DEF 現在的版本。

**評分**:
| 項目 | 分 |
|---|---|
| 紀錄完整度 | 2 |
| 痛點理解 | 2 |
| 未來規劃 | 2 |
| 競爭思考 | 2 |
| 市場/環境 | 2 |
| 商業判斷 | 2 |
| **小計** | **12/12 + 對角線加權 → C5** |

---

## 3. 兩版差在哪(教學重點)

| 看點 | 壞版 | 好版 |
|---|---|---|
| 痛點 | 抄「報價慢」 | 挖出真因(資料散/責任不清/無流程)|
| 未來 | 無 | 分 H2 近 / H3 遠,**抓到沒說出口的「對外透明」** |
| 對手 | 「DEF 有導入」 | DEF 的**盲點** + 我們怎麼跳過它 |
| 我們 | 無 | 能解/不能解、怎麼賺、怎麼產品化 |
| **對角線** | **0 條** | **1 條明寫**(對手現在 × 客戶未來 × 環境過去)|

**一句話**:壞版填了 5 點、得 2 分;好版的價值全在**第 4 段與那條對角線**——那是顧問費的來源。**填滿 ≠ 思考。**

---

## 4. 給學員的練習
拿你最近一場真實會議,先寫壞版(你現在的習慣),再用 `02 模板`改寫成好版,**強迫自己畫出至少一條對角線**,然後用 `03 評估表`自評。交給主管時,只討論:**第 4 段 + 對角線**。
