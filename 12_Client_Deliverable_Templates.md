# 客戶交付物模板(對外版 —— 把內訓思考變成客戶看的東西)

> 前面 02 是**內訓**用(含「我們怎麼賺/壁壘」等內部判斷)。這份是**對外交給客戶**的版本:同樣的思考,但**只露對客戶有價值的部分**,內部商業盤算不外洩。
> 紀律:對外文件不出現內部定價/壁壘盤算、不編造、潛在需求標「待確認」。

---

## A. 客戶會後摘要(Client Recap,會後 24h 內寄)
> 目的:讓客戶覺得「他被聽懂了」,並埋下下一步。

```
【會後摘要 —— {客戶} × {日期}】
1. 我們聽到的重點:{客戶原話重述,證明聽懂}
2. 我們的理解(痛點):您現在的挑戰,核心可能在 {真因},而非只是 {表面症狀}。
3. 值得一起想的下一步:{1-2 個方向,含一個「您可能還沒考慮、但未來會需要」的點}
4. 建議的下一步:{誰、何時、做什麼}
（若有理解不足之處,再請您指正。）
```

## B. 顧問洞察 Memo(Insight Memo,1 頁,展現高度)
> 目的:不只回答客戶問的,給他「沒問到但該知道」的。用 SCQA 收斂。

```
【洞察 Memo:{主題}】
情境(S):您目前 {現況}。
變化(C):但 {對手現在的動作 / 環境的結構轉變} 正在改變遊戲規則。
問題(Q):若只解 {表面問題},未來會 {風險}。
建議(A):我們建議 {對角線結論:對手現在盲點 × 您未來需求 × 環境趨勢 → 所以…}。
為什麼現在:{時間窗 / 先動者優勢}。
（資料來源與假設:{標明哪些已確認、哪些待驗證}）
```

## C. 提案前診斷表(Pre-Proposal Diagnosis,內外混用)
> 目的:提案前先讓客戶確認「我們對你的理解對不對」,降低提案被打槍。

| 項目 | 我們的理解 | 請客戶確認 |
|---|---|---|
| 真正痛點(非表面) | {真因} | ☐ 對 ☐ 要修 |
| 現在最在乎的(JTBD) | {效率/控管/透明…} | ☐ |
| 未來會浮現的需求 | {沒說出口的} | ☐ |
| 成功長什麼樣 | {成效目標} | ☐ |
| 限制/風險 | {資料/人/時程} | ☐ |

## D. 業務 Account Plan(長期經營單一客戶)
```
【Account Plan:{客戶}】
- 客戶軌跡(過去→現在→未來):{用矩陣客戶列}
- 對手在這客戶的滲透:{對手列}
- 環境/法規對這客戶的影響:{環境列}
- 我們的位置:能解什麼 / 下一個切入點 / 怎麼從專案變長期
- 對角線機會:{一條最值得押的}
- 12 個月路線:{Q1-Q4 里程碑}
```
